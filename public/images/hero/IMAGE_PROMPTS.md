## 디자인 가이드라인

### 공통 스타일
1. **색상**: 각 컨텐츠별 지정된 그라데이션 사용
2. **배경**: 어두운 (#0a0a0f ~ #1a1a2f) 계열
3. **효과**: 글로우, 블러, 네온 라이트 효과
4. **분위기**: 신비롭고 프리미엄한 앱 느낌
5. **구도**: 중앙 집중형 또는 좌우 대칭

### 접근 타입별 톤
| 접근 타입 | 색상 톤 | 분위기 |
|----------|--------|--------|
| 무료 (free) | 에메랄드/그린 계열 | 친근하고 접근하기 쉬운 |
| 일일 제한 (daily_limit) | 스카이 블루 계열 | 신선하고 일상적인 |
| 포인트 (points) | 앰버/골드 계열 | 가치 있고 프리미엄한 |
| 프리미엄 (premium) | 퍼플/바이올렛 계열 | 고급스럽고 독점적인 |

### 금지 사항
- 텍스트나 글자 포함 금지
- 현실적인 인물 사진 금지
- 과도하게 복잡한 구성
- 밝은 배경색 사용
- 저품질/픽셀화된 이미지

### 권장 사항
- 추상적이고 상징적인 이미지
- 동양적 모티브의 현대적 해석
- 고해상도 (최소 1600x900px 제작 후 리사이즈)
- 어두운 UI에서 잘 보이는 밝은 포인트 요소

## 이미지 적용 방법

`HeroPage.tsx`에서 이미지를 활성화하려면:

1. 이미지 파일을 `/public/images/hero/` 폴더에 저장
2. 각 ContentCard 컴포넌트의 이미지 영역에 Image 컴포넌트 추가:

```tsx
// ContentCard 컴포넌트 내 이미지 영역
<div className="relative w-full aspect-[16/9] overflow-hidden">
  {/* 플레이스홀더 대신 실제 이미지 사용 */}
  <Image
    src={item.imagePath}
    alt={item.imageAlt}
    fill
    className="object-cover group-hover:scale-105 transition-transform duration-700"
  />
  {/* 오버레이는 유지 */}
  <div className="absolute inset-0 bg-gradient-to-t from-[#0a0a0f] via-transparent to-transparent" />
</div>
```
